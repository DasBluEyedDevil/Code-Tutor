title: "Key Takeaways"
---

*   **Switch Expressions**: Use `->` for concise, readable code.
*   **No Fall-through**: Arrows prevent accidental fall-through bugs.
*   **Return Values**: Switch expressions can return values directly.
*   **Exhaustiveness**: You must cover all cases or provide a `default`.
*   **Pattern Matching**: Switch on types (Java 21+) using `case Type t`.
*   **Guards**: Use `when` to add conditions to patterns.
