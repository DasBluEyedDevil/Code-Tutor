title: "When to Use Each Feature"
---


### Records
- **Use when**: Returning multiple values from functions
- **Use when**: Grouping related data without creating a class
- **Use when**: Creating lightweight, immutable data structures

### Pattern Matching
- **Use when**: Extracting values from complex data structures
- **Use when**: Replacing verbose if-else chains
- **Use when**: Type-checking and casting in one step

### Sealed Classes
- **Use when**: Modeling a fixed set of states (UI state, API results)
- **Use when**: You want exhaustive switch checking
- **Use when**: Creating type-safe state machines
