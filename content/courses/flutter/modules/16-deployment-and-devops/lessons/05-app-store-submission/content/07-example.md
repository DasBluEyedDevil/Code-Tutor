Technical requirements for visual assets:



```text
## Screenshot Sizes (Required)

### iPhone Displays
6.7" (iPhone 14 Pro Max, 15 Pro Max):
- Portrait: 1290 x 2796 pixels
- Landscape: 2796 x 1290 pixels

6.5" (iPhone 11 Pro Max, XS Max):
- Portrait: 1242 x 2688 pixels
- Landscape: 2688 x 1242 pixels

5.5" (iPhone 8 Plus, 7 Plus):
- Portrait: 1242 x 2208 pixels
- Landscape: 2208 x 1242 pixels

### iPad Displays
12.9" (iPad Pro):
- Portrait: 2048 x 2732 pixels
- Landscape: 2732 x 2048 pixels

## Screenshot Specifications

- Format: PNG or JPEG
- Color space: sRGB or P3
- Bit depth: 8-bit
- No alpha channel
- Minimum 72 dpi
- No status bar overlays (or use Apple's status bar)

## App Preview Video Specifications

- Format: H.264, HEVC, ProRes 422
- Resolution: Match device screenshot size
- Frame rate: 30 fps
- Duration: 15-30 seconds
- Audio: AAC, optional
- No letterboxing or pillarboxing

## Creating Screenshot Frames

Recommended tools:
- Figma with device mockup plugins
- Rotato (3D device mockups)
- AppMockUp (web-based)
- Screenshots Pro

Best practices:
1. Capture actual app screens (not mockups)
2. Add device frames for context
3. Include concise captions (5-8 words)
4. Maintain consistent style across all screenshots
5. Localize for each language you support
```
