**Build a Project Initializer Tool!**

You'll create a CLI tool that sets up new Python projects with:
- Proper package structure
- Custom modules
- requirements.txt with popular packages
- Virtual environment instructions
- Example code using third-party libraries

**What you'll practice:**
- Creating packages with `__init__.py`
- Writing reusable modules
- Managing dependencies
- Importing and using modules
- Project organization best practices

**Project Structure:**
```
project_initializer/
├── __init__.py
├── templates/
│   ├── __init__.py
│   ├── web_template.py
│   └── data_template.py
├── utils/
│   ├── __init__.py
│   ├── file_utils.py
│   └── package_checker.py
└── main.py
```

**Features:**
1. Choose project type (web, data science, general)
2. Generate appropriate folder structure
3. Create requirements.txt with relevant packages
4. Generate sample code files
5. Display setup instructions
