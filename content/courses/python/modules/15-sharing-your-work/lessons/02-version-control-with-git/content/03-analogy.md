**GitHub = Remote Git hosting + collaboration tools**

**Key features:**

**1. Pull Requests (PRs)**
- Propose changes
- Code review
- Discussion
- CI/CD checks

**2. Issues**
- Bug reports
- Feature requests
- Task tracking
- Discussion

**3. Forking**
- Copy someone's repo
- Make your changes
- Contribute back via PR

**Standard workflow:**

```
1. Fork repo (your copy on GitHub)
2. Clone fork to your computer
3. Create feature branch
4. Make changes + commit
5. Push to your fork
6. Open Pull Request
7. Address review comments
8. PR gets merged
9. Delete branch
```

**Pull Request description template:**
```markdown
## What
Brief description of changes

## Why  
Why this change is needed

## How
How it was implemented

## Testing
How to test these changes

## Screenshots
(if UI changes)
```

**Code review etiquette:**
- Be respectful and constructive
- Ask questions, don't demand
- Praise good code
- Suggest, don't command
- Focus on code, not person
